# 入学访谈：已有基础与 Mission 锁定

2026-06-28 首次访谈确认的用户起点。用于计算后续课程的 ZPD（最近发展区）。

## 已确立的先验知识
用户声明：**做过一点 LLM 开发**——调过 API、写过 prompt。这意味着 LLM、prompt、token、context window、API call 这些基础术语可视为已知（已收入 [[GLOSSARY.md]] 作为词汇锚点）。

**含义**：课程不应从「什么是 Python / 什么是 prompt」讲起。起点应设在「一次 LLM 调用 → Agent」这一步。第 1 课（什么是 Agent / ReAct 心智模型）即建立在此判断上。

## Mission 双线
1. **求职/转行**（技能导向）→ 课程需产出可上手的代码与可展示的项目，并覆盖面试常考点。
2. **前沿研究**（知识导向）→ 课程需嵌入论文阅读（从 ReAct 起），培养判断力。

## 约束
- 每周 7-15 小时；技术栈 Python；中文授课；偏好理论+动手结合、desirable difficulty。

## Implications（对后续课程的指引）
- 每节课尽量兼顾「能做的技能」+「能读的论文/概念」。
- 第 2 课应为动手课：从零用 Python 实现 LLM + 工具 + 循环，与 ReAct 轨迹对应。
- 框架引入应延后：先用最小依赖讲清原理，再用框架做对比（符合用户「讲清为什么」的偏好）。

_待用户在第 1 课的测验/练习中演示对 Agent、ReAct、Tool use 的理解后，再将这些术语正式收入 [[GLOSSARY.md]]，并写新的学习记录。_
