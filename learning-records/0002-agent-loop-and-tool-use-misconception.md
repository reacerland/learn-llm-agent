# 第 2 课里程碑：Agent 循环 + 一个高频误区

2026-06-29。课程到达「亲手写出最小 Agent 循环」这一里程碑。

## 教学锚点（高价值，需在后续课程持续回扣）
本课刻意钉死一个普遍且致命的认知误区，作为后续判断用户理解深度的探针：

> **误区**：以为「模型调用工具」= 模型自己执行了那个函数。
> **正解**：模型只**请求**（输出一段结构化文字「我想调用工具 X、参数 Y」）；**真正执行函数的是用户的代码**，结果再喂回模型。

**为什么重要**：这是区分「背了八股」与「真正懂 Agent」的分水岭（求职线高频考点）；也是理解后续「工具的权限边界、安全、并行调用、流式」等一切进阶话题的前提。如果用户在第 3 课及以后含糊带过「执行」二字，应回到此点重申。

## 已建立的工程直觉（来自第 2 课代码）
- Agent 的本质是一个「用户拥有的 <code>while</code> 循环」；模型只是循环里的一步。换模型只换 <code>llm()</code>，循环主体不变。
- 「记忆」= <code>messages</code> 数组的不断 append；模型靠它「看见」历史。
- <code>max_steps</code> 是工程兜底，不是摆设——它体现「任务复杂度 = 所需步数」。
