什么是 AI Agent？
从「一次调用」到「自主循环」

你已经会调 LLM 了：输入一段 prompt，得到一段文本。这一课，我们把这一步往前推一格—— 看清「Agent」和「一次调用」之间那道关键的界线。这是后面所有内容的地基。

本节你会获得

01起点：你已经会的，以及它的边界

你做过一点 LLM 开发，所以下面这段你很熟：

response = llm("把这段话翻译成英文：今天天气真好")
# → "The weather is really nice today."

这是一次无状态的、一次性的调用：prompt 进，文本出，结束。它很强大，但有三条硬边界——

  1. 看不见当下：模型只知道你塞进 prompt 的内容，不知道「现在几点」「今天北京几度」。
  2. 做不了事：它只能输出文字，不能真的去查数据库、发邮件、跑一段代码。
  3. 不会自我纠错：如果第一步就错了，它没有「再想想」的机会——因为它只走一步。
记住这个对比 一次 LLM 调用 = 「问答」。Agent 想解决的问题是：当任务需要多步、需要外部信息、需要边做边调整时，怎么办？

02那么，Agent 到底是什么？

一句话：

Agent = 让 LLM 进入一个循环，给它工具和记忆， 使它能自主地朝目标推进，直到完成。

业界最广为引用的拆解，来自 Lilian Weng 的综述（见下方主源）。把它记成一个公式：

Agent 🤖 = LLM 💬 + 规划 📝 + 记忆 🧠 + 工具 🧰

其中，把「一次调用」变成「Agent」最不可或缺的，是循环 + 工具这两样（规划与记忆是进阶强化，后面单独成课）。具体说：

  • LLM 作为大脑：负责推理——「现在该干嘛」。
  • 工具（Tool use）：给它「看」和「做」的能力——查天气、搜网页、跑代码、调 API。
  • 循环 + 观察（Observe）：用完工具，把结果喂回去，让 LLM 决定下一步；如此往复，直到它认为任务完成。
关键洞察 工具补上了「看不见当下、做不了事」；循环补上了「不会自我纠错」。 理解这一点，你就理解了 Agent 和普通调用之间那道线为什么存在。

03ReAct：最经典的 Agent 循环

把上面这套「循环」说清楚、并让它真正能用的，是 2022 年的论文 ReAct （Reasoning + Acting，Yao et al.）。它的循环只有三步，反复执行：

Thought思考：基于当前情况，下一步该做什么、为什么
Action行动：调用某个工具（带参数）
Observation观察：拿到工具返回的结果

看一个具体例子。任务是「今天北京适合穿什么？」——模型本身不知道今天天气，必须借助工具：

Thought我需要知道今天北京的实时天气，才能给穿搭建议。我应该调用天气工具。
Actionget_weather(city="北京")
Observation北京今天 5°C，西北风 6 级。
Thought5°C 加大风，体感很冷，需要保暖外套和防风装备。
Final今天北京 5°C 且大风，建议穿厚外套、围巾，注意防风保暖。
为什么 ReAct 是源头 你后面会遇到的几乎所有 Agent（哪怕是用了 LangChain、Claude、各种新框架），底层都在反复跑这个 Thought → Action → Observation 的循环。把它刻进脑子，等于拿到了 Agent 的「底层语法」。

04关键判断：什么时候用 Agent？

这是 Anthropic 那篇《Building Effective Agents》里最重要、也最容易被忽略的洞察。他们把基于 LLM 的系统分成两类：

  • Workflow（工作流）：步骤是你预先编排好的，LLM 只在被调用的那几步里干活。路径固定。
  • Agent（智能体）：下一步由 LLM 自己决定，它会根据观察动态选择走哪条路、用哪个工具、何时停。
新手最常踩的坑 听到「Agent」就想用 Agent。但 Agent 用 LLM 来做决策，意味着更多不确定性、更高成本、更难调试。 能用固定工作流解决的，就不要上 Agent。先问：这一步，是不是必须由模型自己临场决定？

06动手回忆：写出你自己的 ReAct 轨迹

任务：「帮我查一下《三体》的豆瓣评分，并说一句值不值得读。」
试着在下面写出 Thought → Action → Observation → Final 的完整轨迹（假装有一个 search_web() 和一个 get_rating() 工具）。写完再点开参考。

Thought我需要先确认《三体》在豆瓣上的条目和评分。先搜索。
Actionsearch_web("三体 刘慈欣 豆瓣")
Observation找到豆瓣条目，id=...，《三体》三部曲。
Thought找到条目了，现在去取它的评分。
Actionget_rating(book="三体")
Observation豆瓣评分 8.8 / 10，超百万人评分。
Final《三体》豆瓣评分 8.8，公认的高分科幻，值得一读。

你的版本不必和这一模一样——重点是：每一步都先想（Thought）再动（Action），拿到结果（Observation）再决定下一步。这就是 ReAct。
